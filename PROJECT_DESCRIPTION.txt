================================================================================
                                 OVERVIEW
================================================================================

Sehat AI is an AI-powered medical triage and hospital finder application. It 
helps users describe their symptoms, receive intelligent triage routing, and 
find the nearest appropriate healthcare facility with real-time navigation.

The application does NOT diagnose conditions or prescribe treatments - it 
purely routes users to the appropriate type of specialist and healthcare 
facility based on their symptoms.

================================================================================
                              CORE FEATURES
================================================================================

1. AI-POWERED SYMPTOM ANALYSIS
   - Users describe their symptoms in natural language
   - OpenRouter AI API analyzes symptoms and determines:
     * Appropriate specialist type (Cardiologist, Neurologist, etc.)
     * Hospital department (Cardiology, Emergency, Pediatrics, etc.)
     * Urgency level (normal, urgent, emergency)
     * Facility type needed (clinic, hospital, emergency_room)
     * Search keywords optimized for Google Maps

2. INTELLIGENT HOSPITAL/CLINIC FINDER
   - Uses Google Maps Places API to find nearby medical facilities
   - Searches based on AI-determined specialist/department keywords
   - Calculates distance from user's current location
   - Shows ratings, open/closed status, and contact information
   - Sorts facilities by distance or rating

3. REAL-TIME NAVIGATION
   - Turn-by-turn directions to selected facility
   - Multiple travel modes: Driving, Bicycling, Walking
   - Live location tracking during navigation
   - Distance and estimated travel time display
   - Draggable route info overlay

4. USER AUTHENTICATION & HISTORY
   - Google Sign-In via Firebase Authentication
   - Search history saved to Firebase Firestore
   - Users can view and reload previous symptom searches
   - Persistent user sessions

5. DARK/LIGHT MODE THEMING
   - Full dark and light mode support
   - Theme preference saved to localStorage
   - Consistent theming across all pages

6. RESPONSIVE DESIGN
   - Desktop: Side panel with hospital cards + map view
   - Mobile: Bottom sheet interface with swipe gestures
   - Smooth animations using Framer Motion

================================================================================
                              USER FLOW
================================================================================

1. LANDING PAGE (/)
   - User sees hero section with app description
   - Can sign in with Google account
   - Clicks "Get Started" or "Start Triage"

2. SYMPTOM INPUT (/interaction)
   - User types symptoms in text area
   - Example: "I have chest pain and difficulty breathing"
   - Clicks "Analyze Symptoms"

3. AI TRIAGE RESULT
   - AI returns routing information:
     * Specialist: "Cardiologist"
     * Department: "Cardiology"
     * Urgency: "urgent"
     * Search keywords: ["cardiology hospital", "heart specialist"]
   - User sees triage summary card
   - Clicks "Find Nearby Facilities"

4. HOSPITAL FINDER (/maps)
   - Map shows user location and nearby facilities
   - Facility cards show:
     * Name, rating, distance
     * Open/Closed status
     * Address and phone number
     * Action buttons (Directions, Call, Website)

5. NAVIGATION
   - User selects a facility
   - Gets route with travel time and distance
   - Can start live navigation
   - Multiple travel modes available

================================================================================
                        ENVIRONMENT VARIABLES
================================================================================

Required in .env file:

  VITE_OPENROUTER_API_KEY      # OpenRouter API key for AI triage
  VITE_GOOGLE_MAPS_KEY         # Google Maps API key
  VITE_FIREBASE_API_KEY        # Firebase API key
  VITE_FIREBASE_AUTH_DOMAIN    # Firebase auth domain
  VITE_FIREBASE_PROJECT_ID     # Firebase project ID
  VITE_FIREBASE_STORAGE_BUCKET # Firebase storage bucket
  VITE_FIREBASE_MESSAGING_SENDER_ID
  VITE_FIREBASE_APP_ID

================================================================================
                            SETUP & RUNNING
================================================================================

1. Install dependencies:
   npm install

2. Create .env file with required API keys (see above)

3. Start development server:
   npm run dev

4. Open browser at http://localhost:5173

5. Build for production:
   npm run build
